# Abelian CPU 挖矿用户指南

本文档详细介绍了如何在 Abelian 区块链上进行 CPU 挖矿，包括使用内置 CPU 矿工的独立挖矿和矿池挖矿选项。

请参考 [Abelian 应用程序基础常识](/zh/guide/) 了解软件架构，并选择适合您的挖矿方式（独立挖矿或加入矿池）。

访问 Abelian 官方网站的下载页面，下载适合的挖矿客户端。

所有操作均通过命令行界面（CLI）完成。Linux 用户请打开终端，Windows 用户请打开 PowerShell 或其他您选择的 Shell 应用。

## 概述

Abelian 上的 CPU 挖矿可以通过以下两种方式进行：

- **独立挖矿**：使用您自己的 Abelian 节点和内置 CPU 矿工独立挖矿
- **矿池挖矿**：使用专用的 CPU 挖矿客户端加入矿池

## 前提条件

### 系统要求

| 组件       | 独立挖矿               | 矿池挖矿         |
|------------|------------------------|-----------------|
| **CPU**    | 4+ 核心（推荐 8+ 核心）| 2+ 核心         |
| **存储**   | 160GB+（普通节点）     | 100MB（仅客户端）|
| **内存**   | 8GB+                   | 4GB+            |
| **网络**   | 稳定的宽带连接         | 稳定的宽带连接  |

### 选择挖矿方式

**独立挖矿：**

- ✅ 获得 100% 的区块奖励
- ✅ 支持网络去中心化
- ✅ 使用 `abec` 内置的 CPU 矿工
- ❌ 需要运行 Abelian 节点
- ❌ 奖励不稳定（高波动性）
- ❌ 算力低于 GPU 挖矿
- **最适合**：希望运行节点并支持网络的用户

**矿池挖矿：**

- ✅ 定期、可预测的奖励
- ✅ 无需运行 Abelian 节点
- ✅ 硬件要求较低
- ✅ 优化的 CPU 挖矿客户端
- ❌ 与其他矿工分享奖励
- ❌ 矿池费用（通常为 1-3%）
- **最适合**：资源有限但希望获得稳定奖励的用户

## 独立挖矿设置

### 1. 安装 Abelian 节点

您需要一个启用了 CPU 挖矿功能的 Abelian 节点。请按照 [Abelian 节点 (abec) 手册](/zh/guide/abelian-node) 设置 `abec`。

**`abec.conf` 中 CPU 挖矿的关键配置：**

```ini
# 节点类型
# 普通节点即可满足挖矿需求
nodetype=normalnode

# 启用 CPU 挖矿
generate=1

# 设置您的挖矿奖励地址
miningaddr=[您的钱包地址]
```

### 2. 创建钱包地址

您需要一个钱包地址来接收挖矿奖励。最简单的方法是使用 [Abelian Pro](/zh/downloads/latest#abelian-pro-移动应用)：

1. 下载并安装 Abelian Pro 移动钱包
2. 创建新钱包或导入现有钱包
3. 进入 `个人资料 → 账户详情`
4. 点击地址字段旁边的眼睛图标
5. 复制您的钱包地址

### 3. 启动独立挖矿

进入 `abec` 目录并启动带有 CPU 挖矿功能的节点：

::: code-group

```shell [Windows]
$ .\abec.exe
```

```shell [macOS]
$ sh start_abec.sh
```

```shell [Linux]
$ sh start_abec.sh
```

:::

**替代方案：在运行中的节点上启用挖矿：**

如果 `abec` 已经在运行，您可以使用 `abectl` 启用 CPU 挖矿：

```shell
# 使用 4 个核心启动 CPU 挖矿
$ sh start_abectl.sh setgenerate true 4

# 检查挖矿状态
$ sh start_abectl.sh getmininginfo

# 停止 CPU 挖矿
$ sh start_abectl.sh setgenerate false
```

## 监控与优化

### 性能指标

在 CPU 挖矿时，请监控以下关键指标：

- **算力**：您的挖矿速度（H/s, KH/s）
- **CPU 使用率**：应保持在系统稳定范围内
- **CPU 温度**：保持在 80°C 以下
- **接受的份额**：成功提交的工作（矿池挖矿）
- **拒绝的份额**：应尽可能少（<2%）

### 优化建议

**硬件优化：**

- 确保 CPU 散热良好
- 监控挖矿时的 CPU 温度
- 考虑降压以提高效率
- 使用高性能 CPU 散热方案

**软件优化：**

- 根据 CPU 核心数调整线程数
- 保留 1-2 个核心用于系统操作
- 使用挖矿强度设置平衡性能
- 监控系统响应速度

**系统优化：**

- 关闭不必要的应用程序
- 设置挖矿进程的优先级
- 确保稳定的电源供应
- 在长时间挖矿时监控系统稳定性

### CPU 挖矿性能预期

**不同 CPU 类型的典型算力：**

- **入门级（2-4 核心）**：100-500 H/s
- **中端（4-8 核心）**：500-2000 H/s
- **高端（8+ 核心）**：2000-5000+ H/s

::: warning 性能说明
CPU 挖矿的算力通常远低于 GPU 挖矿。建议将其主要用于支持网络而非盈利。
:::

## 故障排除

### 常见问题

**CPU 使用率过高：**

- 减少挖矿核心数
- 降低挖矿强度
- 确保散热良好
- 监控系统稳定性

**算力过低：**

- 检查 CPU 温度是否导致降频
- 验证线程数设置
- 更新系统和驱动程序
- 确保电源供应充足

**连接问题（矿池挖矿）：**

- 验证矿池服务器地址和凭据
- 检查网络连接
- 尝试不同的矿池服务器
- 确认防火墙设置

**系统不稳定：**

- 降低 CPU 挖矿强度
- 减少线程数
- 检查系统温度
- 确保电源供应充足

- **保持软件更新**至最新版本
- **监控挖矿活动**，发现异常行为

### 操作安全

- **从低强度开始**，测试系统稳定性
- **持续监控温度**，避免过热
- **准备好散热方案**后再长时间挖矿
- **计划系统维护**和更新

## 经济性与盈利

### 影响盈利的因素

- **CPU 算力**：您的挖矿速度
- **网络难度**：每 200 个区块调整一次
- **区块奖励**：当前为 64 ABEL（时代 2）
- **电力成本**：主要运营支出
- **CPU 效率**：每瓦特算力

### 收益计算

**独立挖矿：**

- 收益 = (您的算力 / 全网算力) × 区块奖励 × 每日区块数
- 波动性极高，可能长时间无法找到区块

**矿池挖矿：**

- 收益 = 您的贡献 × 矿池奖励 × (1 - 矿池费用)
- 波动性较低，定期获得小额收益

::: tip 盈利现实
由于算力低和电力成本高，CPU 挖矿通常无法盈利。建议将其作为支持网络和学习挖矿的方式。
:::

使用在线计算器并在 [AbelScan](https://explorer.pqabelian.io) 上监控网络统计数据。

## 高级主题

### 多核优化

- 为每个 CPU 配置最佳线程数
- 使用 CPU 亲和性绑定核心
- 平衡挖矿与系统响应速度

### 混合挖矿设置

- 在同一系统中结合 CPU 和 GPU 挖矿
- 优化矿工之间的资源分配
- 监控总系统功耗

### 矿场 CPU 集成

- 在 GPU 矿机上使用 CPU 挖矿
- 实施集中监控
- 优化混合设置的电源和散热
